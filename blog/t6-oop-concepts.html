Object Oriented Programming concepts

Variable Scope

July 24,2015

Before I go on to today's topic on OOP concept of Variable scope I would like to briefly touch on what OOP terms of class, object and instance are to better understand how variables work in OOP. An object is a software bundle of related state and behavior. A class is a blueprint or prototype from which objects are created. An instance is a single and unique unit of a class. We have a blueprint (class) represents student (object) with fields like name, age, course (class member or variables). And we have 2 students here, Nelly and Nector. So, Nelly and Nector are 2 different instances of the class (Student class) that represent object (Student people). Still muddy?? lets illustrate with a better example.

Let's say you bought some chairs from ikea and you are building them.

Class => The diagram that shows how to build a chair and put it together corresponds to a software class.

Object => Let's say you build five chairs according to the pattern in that diagram. Likewise, you could construct five software objects according to the pattern in a class.

Instance => Each chair has a unique number burned into the bottom of the seat to identify each specific chair. Chair 3 is one instance of a chair object. Likewise, memory location 3 can contain one instance of a software object.

So, an instance (chair 3) is a single unique, specific representation of an object (a chair).

Here we are going to talk about variables and their scope in an OOP paradigm. Well don't get too overwhelmed, its not as wierd as it sounds. Here it goes..

What are Variables?

In computer programming, a variable is a storage location in the memory which has a descriptive name (an identifier) and contains some known or unknown quantity of information referred to as a value. It is helpful to think of variables as containers that hold information.Their sole purpose is to label and store data in memory.

Types of variable

There are 5 types of variables.

constant [A-Z]-capital letters only available throughout application
global variable name starts with the dollar sign ($) These variables are available throughout your entire app, overriding all scope boundaries
class variable name starts with 2 @ sign (@@) Class variables must be initialized at the class level, outside of any methods. They can then be altered using class or instance methods
instance variable name starts with 1 @ sign (@) These variables are available throughout the current instance of the parent class. Instance variables can cross some scope boundaries, but not all of them.
local [a-z] or _ - only small leters or underscore "_" most common variables you will come across and obey all scope boundaries.

Scope of Variables

Each type of variable has a scope. Scope defines where in a program a variable is accessible. There are two types of pseudo-variables not mentioned before. They are nil and self. nil is assigned to an uninitialized variable and self which refers to currently executing object. For this blog, I will not be covering scope of pseudo-variables.

CONSTANT : The value of a constant variable remains constant for the duration of the entire program. While most programming languages do not allow you to change the value assigned to a constant, Ruby does. It will however throw a warning letting you know that there was a previous definition for that variable. Just because you can, doesn't mean you should change the value. In fact, you should not. Constants cannot be declared in methods, and are available throughout your application's. Although constants should not be changed, you can modify the internal states of the objects they reference. 

                CONST1 = "DBC"
                CONST2 = CONST1
                CONST1[0] = "J" # alter string referenced by constant
                puts CONST1 # displays JBC
                puts CONST2 # also displays JBC

GLOBAL : Global variables in Ruby are accessible from anywhere in the Ruby program, regardless of where they are declared. It is useful to know, that a number of pre-defined global variables are available to you as a Ruby developer to obtain information about the Ruby environment. A brief summary of each of these variables is contained in this table.

                $message = "Have a great day!"

CLASS VARIABLES: A class variable is a variable that is shared amongst all instances of a class. This means that only one variable value exists for all objects instantiated from this class. If one object instance changes the value of the variable, that new value will essentially change for all other object instances. Another way of thinking of class variables is as global variables within the context of a single class. A good way to use class variables is when the variable holds a boolean value. Check the example as below. 

        class Animal
          @@is = true
          @@count = 0
          def initialize name
            @type = name
          end
          def to_s
            "This is #{@type}"
          end
          def does_exist?
             @@is
          end

          def count
             @@count += 1
          end
         end
    
        b1 = Animal.new "Dog"
        b2 = Animal.new "Cat"
        b3 = Animal.new "Mouse"

        p b1, b2, b3

        p b1.does_exist?
        p b2.does_exist?
        p b3.does_exist?
        p b1.count
        p b2.count
        p b3.count
        p b1.count
      
      #output
      #print b1, b2 b3
      #
      #
      #
      #does_exit?
        true
        true
        true
      #count
        1
        2
        3
        4

From the above code we understand that, the first three messages are unique. The strings are stored in the instance variables of the objects. The true value is the value of the class variable, which is called three times. The count value increases by one each time it is called.

Class variables are known to have some perhaps surprising behavior when you look at inheritance. 

                class DemoClass
                   @@my_var = nil
                   def initialize
                       @@my_var = "hello world"
                   end
                   def my_var
                       puts @@my_var
                   end
                end

                class Demo2Class < DemoClass
                   def initialize
                       @@my_var = "goodby world"
                   end
                end

                demo1 = DemoClass.new
                demo1.my_var
                demo2 = Demo2Class.new
                demo2.my_var
                demo1.my_var

                # Output:
                #hello world
                #goodbye world
                #goodbye world

The change to my_var in Demo2Class has bubbled up to our base class. So it is better to avoid class variables in such cases.

INSTANCE VARIABLES : An instance variable belongs to that one instance of the class. It is denoted by a single at symbol (@), and can be accessed from any instance of a class from within a method. They have private access. You can access these variables in any method inside the class. For example if a class contains an instance variable called @count, if one instance of the object changes the current value of @count the change is local to only the object that made the change. Let us see what happens if the count variable declared as a class variable in the previous example is decalred as an instance variable.

    class Animal
        @@is = true
        def initialize name
          @type = name
          @count = 0
        end
        def to_s
          "This is #{@type}"
        end
        def does_exist?
            @@is
        end
        def count
            @count += 1
        end
    end
    
        b1 = Animal.new "Dog"
        b2 = Animal.new "Cat"
        b3 = Animal.new "Mouse"

        p b1, b2, b3

        p b1.does_exist?
        p b2.does_exist?
        p b3.does_exist?
        p b1.count
        p b2.count
        p b3.count
        p b1.count
       
        #output
        #print b1, b2 b3
        #
        #
        #
        #does_exit?
        true
        true
        true
        #count
        1
        1
        1
        2

Thus we can understand from this that for each instance (b1,b2,b3) of the objects that belongs to the same class have their own local copies of the variable which are independent of changes made in any other instances.

LOCAL : The local variables are the normal variables, example x = 25, y = gaurish where x and y are local variables. They only exists within it's scope (block in which it is declared) They may be local to a function, or local to the class declaration, which is different than being a class variable. Locals in a class declaration go out of scope when the class declaration ends. 

                 def test1
                     var1 = 2
                     var1.times do
                         var2 = 3
                         p "v1=#{var1} v2=#{var2}"
                     end
                   p "v1=#{var1}"
                   p "v2=#{var2}"  # Out of scope
                 end

As you can see in the above example, var2 was initialized witin the .times loop and as soon as it hit end var2 disappears. Hence it will throw an error as that variable cannot be accessed.
