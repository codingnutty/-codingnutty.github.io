Ruby Classes

Modeling a real world object

July 18,2015

Ruby is known to be one of the most powerful object oriented languages(OOP). A newbie may wonder what object oriented programming is? It is a programming model based on the concept that everything around you is an object. An object in programming can be a variable, a datastucture or a function. For example, When one says Brian is speaking. Brian here is an object who performs the function of speaking. This can be translated into a programming paradigm.

In OOP there exists a class. A class is considered as a code-template for creating these objects,providing some values to them and implementation of their behaviour(function). Think about a modern digital camera as an object. You take pictures with it - it captures images and stores them. They also let you do things with the stored images, like cropping and stitching. The object camera, can be represented as classes.You construct a class by writing what looks a lot like an outline. There are rules that you must follow to write this outline in the Ruby language and many other languages. In computer programming these rules are called the language's syntax. For a larger real world example, A bank account can be considered as a class, The attributes or values in it can be considered as the Account Number,Bank name, Owner Name, Account Balance etc and the operations or functions to be performed are Withdraw, Deposit, Generate Statement. The class bank account handles all of these within it and enables a cleaner environment for operations to take palce.

Okay! Now that we have got a basic idea of what classes are, lets delve into what classes have. We have learned that classes are objects as objects are way to represent a class. Each object can have a class and instance variables. Before you question what an instance variable is? To put all of this into perspective let us consider an example.

Consider an Animal. We can create an object called Animal1,which is considered as creating an instance of class Animal. Animal1 has some properties like type, gender and food preference. What are the functions it performs? It can walk and eat. So let us consider a class Animal. One can declare a class using the "class" keyword.

        class Animal
        end

Let us now set instance variables for the class. Each variable will be the properties that animal1 embodies. Each instantiated object in a class can store different values into these variables. Every instance of a class Animal will have certain properties like we discussed above. The variables will have an @ sign in front of them which denotes that it an instance variable. 

         class Animal
                @type = nil
                @gender = nil
                @size = nil
         end

Note that the name of the class begins with a capital letter. This is usually the denotation for the class name.

Setting these variables to nil does not provide any value to the object created. So in order for the object to access these properties, we need to use a constructor method or function which contructs these properties. Here the constructor method is called Initialize. As the name says it initialized the instance variables for the object to catch in order to perform required functions. 

            class Animal
                def initialize(type,gender,food_pref)
                    @type = type
                    @gender = gender
                    @food_pref = food_pref
                end
            end

The above code assigns the object specific values passed while creating the object to these instance variables.

Now that we have the properties set, we need to set the funtions performed by this animal. The funtions performed by the animal are walking and eating. 

           class Animal
                def initialize(type,gender,food_pref)
                    @type = type
                    @gender = gender
                    @food_pref = food_pref
                end

                def walking
                   puts "The #{@gender} #{@type} is walking in the park."
                end

                def eating
                   puts "The #{@gender} #{@type} likes to eat #{@food_pref}."
                end
           end

In order for these method to function we need to create an object for this class and pass values that can be stored in the instance variables.These values are called arguments. Note that this class requires 3 arguments to be passed as there are 3 variables waiting to accept them. An arument error is raised if there insufficient number of arguments are passed while creating a new object. 

      animal1 = Animal.new("Dog","male","meat")
      animal1.walking  => "The male Dog is walking in the park." 
      animal1.eating  => "The male Dog likes to eat meat." 
      animal2 = Animal.new("cat","female","mouse")
      animal2.walking  => "The female cat is walking in the park." 
      animal2.eating  => "The female cat likes to eat mouse."

I hope the above examples have provided a clear understanding of the uses of a class. I was able to create and store properties for 2 different animals that paerform the same function. The class here provided a platform for the animals to perform the fucntions.

Hope this helped. One can use this example as a platform to represent interaction of real world objects via ruby code.
