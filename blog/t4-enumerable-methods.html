Ruby Explained- Enumerating Enumerable:

Enumerable#map/#collect

July 09,2015

This week was a dive into Ruby Arrays,Hashes Iterators and Enumerators. Solved some problems which were more challenging than before.For this tehcnical blog, I am about to explain one of the Ruby Enumerable methods called "map"/"collect".

What is the difference between map and collect methods? is a question that often raises amongst we beginner ruby programmers.They are pretty much the same.map is implemented in C which is the low level language that is more readable to the machine as rb_ary_collect and enum_collect. The C level implementation of the 2 methods is a true explanation why they are same. Refer collect and map on API Docs.

#collect/#map on Arrays

Lets us understand what is returned when a block is provided and when it is not.

  1. ary.collect {|item| block } → new_ary
  2. ary.map {|item| block } → new_ary
  3. ary.collect → an_enumerator
  4. ary.map → an_enumerator
map and collect evaluates the code block and passes each item to the block, where the block operation is performed aon the item passed and result of the block is added to a new array. At the end an array containing the values returned by the code block is returned. If no block is provided the methods call the implied block which is : {|item| item}.The resulting array is the same as the given array.

a = [1,11,22,33,44,55,66,77,88,99]
#=> [1,11,22,33,44,55,66,77,88,99]

b = a.collect{|num| num.even?}
#=> [false,false,true,false,true,false,true,false,true,false]

c = a.map{|num| num.even?}
#=> [false,false,true,false,true,false,true,false,true,false]

#collect/#map on Hashes

When map and collect methods are used on a hash and a block is a provided, they pass each key-value pair through the block and returns - an array containing a set of two-item arrays, one for each key value pair, arr[0] being key and arr[1] being value of each array. This method can be used to convert hashes to arrays.

hash = { :a => 1, :b => 2, :c => 3 }

hash.map {|k, v| v*2 } # => [6, 2, 4]

hash.collect {|k, v| v*2 } # => [6, 2, 4]

hash.map => [["a", 1], ["b", 2], ["c", 3]]

In Ruby difference between collect and map does not exist.C level implementation of these methods shows that they both are same. Collect is an alias to Map
